## 数据源简介

### 什么是法答网？

法答网是由**最高人民法院**建设和管理的全国法院系统内部信息共享平台，旨在解决审判一线的法律适用疑难问题。其核心理念是 **"有问题，找法答"**。

法答网的运行机制如下：

```
基层/中级/高级法院干警
        │
        │  提交法律适用问题
        ▼
    法答网平台
        │
        │  分配给相关业务领域专家
        ▼
    答疑专家（资深法官/学者）
        │
        │  在规定时限内提出答疑意见
        │  经业务部门负责人审核
        ▼
    答疑意见发布
        │
        │  优秀咨询答疑被评选出来
        ▼
 《人民法院报》"法答网精选答问"栏目
        │
        │  向社会公开发布
        ▼
    本仓库收录
```

### 精选答问栏目

自 2024 年 2 月 29 日起，《人民法院报》开设 **"法答网精选答问"** 专栏，将具有典型性、前沿性或疑难复杂的法律适用咨询答疑定期公开发布。截至 2026 年 2 月，已发布 **37 批** 精选答问。

每批次通常包含 **4-10 个问题**，每个问题由以下部分组成：

| 组成部分 | 说明 |
|---------|------|
| **问题标题** | 来自审判一线干警的实际咨询 |
| **答疑意见** | 由最高法院业务部门专家撰写，经审核后发布 |
| **咨询人** | 提出问题的一线法官/干警单位与姓名 |
| **答疑专家** | 答疑的最高法院法官或业务专家 |
| **点评意见**（部分批次） | 由法学教授/知名学者对答疑进行学理点评 |
| **点评专家**（部分批次） | 参与点评的学者信息 |

### 涵盖的法律领域

问题覆盖中国法律体系的多个核心领域：

- **民商法** — 合同纠纷、公司治理、股权转让、融资融券、破产程序
- **刑法** — 受贿罪、滥用职权罪、罪数形态、涉案财产处理
- **行政法** — 工伤认定、行政保全、诉讼时效
- **婚姻家庭法** — 离婚抚养权、财产分割
- **劳动法** — 劳动关系认定、新业态用工
- **知识产权法** — 软件著作权、商业秘密
- **环境法** — 生态损害赔偿磋商
- **执行程序** — 以物抵债、到期债权执行、追加被执行人
- **诉讼程序** — 诉讼时效中断、管辖、证据规则

## 选择该数据源的理由

在选择法律知识数据源用于学习与研究时，法答网精选答问相比其他法律数据源具有以下**六项核心优势**：

### 1. 🏛️ 权威性 — 最高司法机关背书

> 法答网由**最高人民法院**直接管理，答疑意见须经相关业务部门负责人审核同意后方可发布，最大限度保障答疑意见的准确性和权威性。

与其他法律数据源（如裁判文书网的单案判决、学术论文的个人观点、法律博客的非正式解读）相比，法答网的每一条答疑都代表了最高司法机关对特定法律适用问题的**官方立场**。这种权威性是其他任何公开数据源难以比拟的。

### 2. 🎯 针对性 — 直面审判一线痛点

> 问题全部来源于全国四级法院干警在**真实审判工作**中遇到的法律适用难题，而非学者假想的理论问题。

这使得精选答问具有极强的实践导向——每一个问题背后都可能对应着全国法院正在审理的数百甚至数千件类似案件。对于法律从业者而言，这种"从实践中来、到实践中去"的问答具有直接的参考价值。

### 3. 📐 统一性 — 促进法律统一适用

> 法答网的核心功能之一就是**统一法律适用标准**，减少"同案不同判"现象。

在中国这样一个地域广阔、案件量巨大的司法体系中，法律适用的统一性是长期面临的重要挑战。法答网的答疑意见为全国法院提供了可参照的统一标准，是理解中国司法实践主流观点的重要窗口。

### 4. 🔬 典型性 — 精选具有普遍意义的问题

> 并非所有咨询都会被发布——只有经过评选的**具有典型性、前沿性或疑难复杂**的优秀咨询答疑才会通过《人民法院报》公开发布。

这确保了数据集的高质量：每一批次的每一个问题都经过了层层筛选，代表了该领域内最有价值、最值得关注的法律争议。

### 5. 📈 持续性与完整性 — 覆盖两年的连续发布

> 自 2024 年 2 月至今，保持了**约每两周一批**的稳定发布节奏，形成了完整的时序数据。

这为研究中国司法实践中的法律观点演变、热点问题转移提供了宝贵的时间序列数据。截至采集日，37 批数据完整覆盖，无缺失批次。

### 6. 🔓 公开可获取 — 合法合规的数据采集

> 所有精选答问均通过《人民法院报》（rmfyb.com）**向社会公开发布**，内容不涉及具体案件信息，采集和使用不涉及隐私或保密问题。

这确保了数据采集的合法性和合规性，适合用于法律研究、教学和知识库建设。

## 数据采集方法

### 总体架构

数据采集采用 **Python 3.8+** 编写的自动化爬虫脚本，整体流程分为四个步骤：

```
┌─────────────────────────────────────────────────────────┐
│                  fetch_fadawang.py v3.0                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  [1/4] 搜索              [2/4] 抓取详情                │
│  ┌─────────────┐        ┌──────────────┐               │
│  │ rmfyb.com   │        │ 逐个批次      │               │
│  │ 搜索"法答网" │ ────→  │ 请求页面      │               │
│  │ 翻页获取     │        │ 提取正文      │               │
│  │ 全部链接    │        └──────────────┘               │
│  └─────────────┘               │                        │
│                                ▼                        │
│  [3/4] HTML→MD           [4/4] 保存                    │
│  ┌──────────────┐        ┌──────────────┐               │
│  │ 清理HTML标签  │        │ 写入 md/ 目录 │               │
│  │ 识别问答结构  │ ────→  │ 更新元数据    │               │
│  │ 生成 Markdown │        │ .fadawang_   │               │
│  └──────────────┘        │ meta.json    │               │
│                          └──────────────┘               │
└─────────────────────────────────────────────────────────┘
```

### 数据源网站

- **目标网站**：人民法院报官网 `https://www.rmfyb.com`
- **搜索接口**：`/search.html` — 关键词 "法答网"，按标题搜索，时间倒序排列
- **详情页面**：`/content/{YYYYMM}/{DD}/article_{id}.html` — 每批次一个独立页面

### 第一步：搜索与链接发现 (`search_all_batches`)

**技术细节：**

1. **分页搜索**：通过 `search.html` 接口，传入参数：
   - `wd=法答网` — 搜索关键词
   - `search_range=1` — 限定标题搜索（避免匹配正文中提及"法答网"的无关文章）
   - `order=2` — 按发布时间倒序排列
   - `p={page}` — 页码参数，从第 1 页开始遍历

2. **总页数检测**：从首页 HTML 中提取 JavaScript 变量 `total_page_num`，确定搜索结果的页数范围。

3. **链接提取**：使用正则表达式匹配搜索结果页面的 HTML 结构：
   ```python
   # 结构: <a href="/content/..."> <div class='content_item'>
   #   <div class='item_title'>...</div> ... </div> </a>
   result_blocks = re.findall(
       r'<a\s+href="(/content/[^"]+)"[^>]*>(.*?)</a>',
       html_content, re.DOTALL
   )
   ```

4. **批次号识别**：通过 `extract_batch_number()` 函数从标题中提取批次号，支持中文数字转阿拉伯数字：
   - 中文数字映射：一→1, 二→2, …, 十→10
   - 复合数字处理：如 "二十一" → 21, "三十五" → 35
   - 阿拉伯数字直读：如 "12" → 12

5. **去重与过滤**：
   - 标题必须包含 "法答网精选答问"
   - 同一批次号只保留首次出现的链接
   - 提取发布日期（格式：`YYYY-MM-DD 07版`）

### 第二步：批次详情抓取 (`fetch_batch_detail`)

**技术细节：**

1. **HTTP 请求配置**：
   ```python
   HEADERS = {
       "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) "
                     "AppleWebKit/537.36 ... Chrome/120.0.0.0 ...",
       "Accept": "text/html,application/xhtml+xml,...",
       "Accept-Language": "zh-CN,zh;q=0.9,en;q=0.8",
   }
   ```

2. **重试机制**：每个请求最多重试 3 次，间隔时间递增：
   ```python
   for attempt in range(3):
       try:
           resp = requests.get(url, ...)
           resp.raise_for_status()
           return resp.text
       except requests.RequestException:
           time.sleep(DELAY * 2)  # 重试间隔加倍
   ```

3. **编码处理**：自动检测页面编码，优先使用 `apparent_encoding` 识别 GBK 编码的页面，确保中文内容不乱码：
   ```python
   if resp.apparent_encoding and "gb" in resp.apparent_encoding.lower():
       resp.encoding = "gbk"
   ```

4. **内容提取**：
   - 页面标题：从 `<div class="title_nav_text">` 提取
   - 正文内容：从 `<div class="conten_text_box">` 提取，使用多层后备正则匹配以应对页面结构变化

5. **请求间隔**：每次请求之间间隔 1.5 秒（`DELAY = 1.5`），避免对目标服务器造成过大压力。

### 第三步：HTML 到 Markdown 转换 (`html_to_markdown`)

这是整个流程中最核心、最复杂的步骤。转换策略如下：

**A. HTML 清理**

```python
# 1. 移除图片标签 <img .../>
text = re.sub(r'<img[^>]*/?>', '', raw_html)

# 2. 移除 font/div 标签但保留文本内容
text = re.sub(r'</?font[^>]*>', '', text)
text = re.sub(r'</?div[^>]*>', '', text)

# 3. <br> 和 <br/> 替换为换行符
text = re.sub(r'<br\s*/?>', '\n', text)

# 4. 解码 HTML 实体（&amp; → &, &lt; → <, 等）
text = html.unescape(text)

# 5. 处理特殊空白字符
text = text.replace(' ', ' ').replace('　', ' ')  # &nbsp; 和全角空格
```

**B. 结构化识别**

脚本通过正则表达式识别每行的语义角色，并应用对应的 Markdown 格式：

| 识别模式 | Markdown 格式 | 示例 |
|---------|--------------|------|
| `问题N：...` | `## 问题N：...` | `## 问题1：网络主播为公司带货，双方是否存在劳动关系？` |
| `答疑意见：...` | `**答疑意见：** ...` | 加粗标签后接正文 |
| `答疑专家：...` | `*答疑专家：...*` | 斜体标注专家信息 |
| `咨询人：...` | `*咨询人：...*` | 斜体标注咨询人信息 |
| `点评意见：...` | `**点评意见：** ...` | 加粗标签后接正文 |
| `点评专家：...` | `*点评专家：...*` | 斜体标注专家信息 |
| `开栏的话` | `> **开栏的话**` | 引用块格式（仅第一批次） |
| 开栏内容 | `> 段落文本` | 引用块格式 |

**C. 页面头部元数据**

每个 Markdown 文件顶部包含结构化的元数据：

```markdown
# 法答网精选答问（第XX批）

> **来源：** 人民法院报 (rmfyb.com) - 法答网精选答问
> **链接：** https://www.rmfyb.com/content/.../article_....html
> **发布日期：** YYYY-MM-DD
> **抓取时间：** YYYY-MM-DD HH:MM:SS
```

**D. 后处理**

```python
# 合并连续空行（3个以上 → 2个）
markdown = re.sub(r"\n{3,}", "\n\n", markdown)

# 清理空引用行
markdown = re.sub(r"\n>\s*\n>", "\n>", markdown)
```

### 第四步：增量更新与元数据管理

**元数据文件 (`.fadawang_meta.json`)：**

```json
{
  "version": 3,
  "source": "rmfyb.com",
  "last_run": "2026-06-27 19:58:42",
  "processed_batches": {
    "1": {
      "title": "法答网精选答问（第一批）",
      "url": "https://www.rmfyb.com/content/202402/29/article_...html",
      "publish_date": "2024-02-29",
      "md_file": "法答网精选答问（第01批）.md",
      "fetched_at": "2026-06-27 19:58:40"
    }
  }
}
```

**增量更新逻辑：**

```python
def get_new_batches(batches, meta, force=False):
    if force:
        return set(batches.keys())         # 强制全量
    processed = {int(k) for k in meta.get("processed_batches", {}).keys()}
    return set(batches.keys()) - processed  # 仅新批次
```

这确保了：
- 默认运行时仅抓取网站新增的批次（增量更新）
- 已抓取的内容不会被重复请求和处理
- 元数据文件保存了完整的抓取历史记录

### 兼容性与鲁棒性设计

| 设计点 | 实现方式 |
|-------|---------|
| **编码兼容** | 自动检测 GBK/UTF-8，优先使用 `apparent_encoding` |
| **网络容错** | 3 次重试，指数退避间隔 |
| **页面结构变化** | 多层后备正则匹配 |
| **中文数字批次号** | 完整的中文数字解析器（一→100+） |
| **Windows 终端** | 自动 `reconfigure` stdout/stderr 为 UTF-8 |
| **请求频率控制** | 1.5 秒固定间隔，重试间隔加倍 |

### 采集脚本用法

